事务 (一组要么同时执行成功,要么同时执行失败的sql语句.是数据库操作的一个执行单元)

  • 原子性
    表示一个事务内的所有操作是一个整体,要么全部成功,要么全部失败
  • 一致性
    表示事务内如果有一个操作失败, 所有更改过的数据都必须回滚到修改前的状态
  • 隔离性
    事务查看数据时数据所处的状态，要么是另一个事务修改它之前的状态，要么是另一个事务修改它之后的状态，事务不会查看中间状态的数据。
    • 读取未提交
    • 读取已提交
    • 可重复读
    • 序列化
  • 持久性
    持久性的事务完成之后，它对系统的影响是永久性的
